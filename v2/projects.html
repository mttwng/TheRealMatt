Hackathons


VSee (LA Hacks)

VSee is a search engine designed to help startups more easily find Venture Capital (VC) Firms. VSee's information was scraped from Venture Capital sights, such as cbinsights.com. This information was then stored into an Elasticsearch Django stack run on an Amazon EC2 server. With this we developed an algorithm based on the location, funding round, and market of a startup to match them with VC's. The goal was to make it easier for startups to find potential funding.


Introduction


Hog

Hog is a dice game played between two players in which the players choose how many dice to roll baed on optimal strategy. The goal for this project was to write a program which won more than 60% of the time against a random computer based on proper selection of dice to roll.

Trends

Trends was a project based on Twitter Tweets around the US. We parsed through Tweets to find trends in the data. We primarily compared geographic location data to sentiment data from tweets to determine states with different overall positive and negative sentiments.

Ants

Ants is a tower defense game, wherein ants protect a tower against bees. We designed varying algorithms to enable different abilties for each of the characters in the game. The project tied in several object-orientated programming paradigms, and involved understanding testing a large program with many related parts.

Scheme

In this project we created a Scheme interpreter for a subset of the Scheme language using Python. We also implemented some small Scheme programs to run with this interpreter.


Data Structures


Ocean

Ocean was a simulation of an ocean containing fish and sharks. Both fish and shark had several properties based on their surrondings and species of whether they would survive or not. We also wrote code to convert the entire ocean into a run-length encoding.

Network

In this we implemented a program which plays the game Network against a human or other computer program. Network is an 8x8 board game played to create a "Network" across the board. This project was heavily based around game theory, breadth, and depth first search algorithms.

Weighted Undirected Graphs and Minimum Spanning Trees

The purpose of this project was to become comfortable with applications that combine a variety of data structures and algorithms in a big ol' mess of references pointing just every which way. Using past data structures (Disjoint Sets and Linked Lists), we created a graph and tree data structure with functions for usability such as a neighbor() function.


Machine Structures


MapReduce

We used a MapReduce Strategy to answer a big data problem. We calculcated co-occurences amoung words in documents. The end result of this project was run on Amazon EC2 servers using Hadoop.

MIPS Instruction Set Emulater

We created an instruction interpreter for a subset of MIPS code. This project allowed us to become more familiar with MIPS and C programming.

Image Processing and Optimization

The goal was to optimize a formula for a 2D convultion on images. The tricks we used for the optimization included Register Blocking, Loop Ordering, SSE Intructions, Loop Unrolling, OpenMP, and Compiler Tricks.

Processor Design

Using Logism, we created a 16-bit two-cycle processor similar to MIPS. We designed an Arithmetic Logic Unit, a Register, and the CPU for this project.
